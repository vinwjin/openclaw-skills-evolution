# OpenClaw Skills Evolution v0.6.3

> GitHub Release notes / GitHub Releases 页面文案

## 中文

本次版本聚焦于 **compaction 接入稳定性、OpenClaw host 能力复用** 与 **发布链路整理**。  
核心目标是让 `skills-evolution-compactor` 在真实 OpenClaw 环境中更自然地复用当前会话的 provider、模型与鉴权能力，同时补齐文档、测试与打包元信息，保证 GitHub / npm 发布内容一致。

### ✨ Highlights

- **优先复用 OpenClaw 当前会话的鉴权与模型能力**
  - compaction provider 现在会优先读取 OpenClaw host/runtime 提供的默认模型、provider 配置与鉴权信息。
  - 当 host 可提供鉴权时，会优先走 OpenClaw 当前会话链路；无法获取时再回退到插件自身的 `apiKeyEnv` 配置。

- **补齐双 API 传输适配**
  - 摘要子进程现已支持：
    - `openai-completions`
    - `anthropic-messages`
  - 会根据配置自动切换对应的 endpoint、headers 与 request body 结构，提升不同 provider 下的兼容性。

- **发布内容与版本元信息同步**
  - 同步更新插件版本到 `0.6.3`
  - 补充 `CHANGELOG.md`
  - 将 changelog 纳入 npm 打包清单
  - README 补齐 0.6.3 发布说明与配置说明

### 🔧 Changes

- compaction provider 注册时传入 OpenClaw runtime / host config
- 新增 `preferOpenClawAuthModel` 配置项
- 新增 `api` 配置项，用于声明摘要调用的 API 传输类型
- 改进 compaction 相关说明文档与版本记录
- 新增针对 host 鉴权桥接与传输适配的回归测试

### ✅ Verified

- `node tests/plugin.test.js`
- `npm pack --dry-run`

### 📦 Installation

```bash
openclaw plugins install @vinwjin/openclaw-skills-evolution
```

或：

```bash
npm install -g @vinwjin/openclaw-skills-evolution
```

### ⚠️ Notes

如需让本插件真正接管 OpenClaw 的 compaction，请确认在 `openclaw.json` 中启用了：

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "provider": "skills-evolution-compactor"
      }
    }
  }
}
```

感谢使用 OpenClaw Skills Evolution。  
如果你正在做真实会话压缩验证，这个版本会更适合作为 host 鉴权 / 模型复用链路的稳定基线。

## English

This release focuses on **compaction integration stability**, **reusing OpenClaw host capabilities**, and **cleaning up the release pipeline**.  
The main goal is to make `skills-evolution-compactor` reuse the active session's provider, model, and authentication path more naturally inside real OpenClaw environments, while keeping GitHub and npm release artifacts aligned.

### ✨ Highlights

- **Prefer OpenClaw host auth and model capabilities**
  - The compaction provider now prefers the default model, provider config, and auth context exposed by the OpenClaw host/runtime.
  - When host-level auth is available, the plugin follows the active OpenClaw session path first and only falls back to its own `apiKeyEnv`-based configuration if needed.

- **Dual API transport support**
  - The summarizer subprocess now supports:
    - `openai-completions`
    - `anthropic-messages`
  - It switches endpoint, headers, and request body shape automatically based on config, improving compatibility across providers.

- **Release metadata cleanup**
  - Version metadata is synced to `0.6.3`
  - `CHANGELOG.md` has been added
  - The changelog is now included in the npm package contents
  - README release notes and config guidance are updated for 0.6.3

### 🔧 Changes

- Pass OpenClaw runtime / host config into the compaction provider at registration time
- Add the `preferOpenClawAuthModel` config option
- Add the `api` config option to declare the summarizer transport mode
- Improve compaction-related documentation and version notes
- Add regression coverage for host-auth bridging and transport adaptation

### ✅ Verified

- `node tests/plugin.test.js`
- `npm pack --dry-run`

### 📦 Installation

```bash
openclaw plugins install @vinwjin/openclaw-skills-evolution
```

or:

```bash
npm install -g @vinwjin/openclaw-skills-evolution
```

### ⚠️ Notes

To let this plugin fully take over OpenClaw compaction, make sure the following is enabled in `openclaw.json`:

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "provider": "skills-evolution-compactor"
      }
    }
  }
}
```

Thanks for using OpenClaw Skills Evolution.  
If you're validating real session compaction, this release is a better baseline for host-auth and model-reuse behavior.
