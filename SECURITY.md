# Security Policy

---

## 一、敏感信息处理

### 禁止硬编码

- [ ] API keys / tokens / passwords
- [ ] 用户凭证
- [ ] 私钥文件路径

### 脱敏要求

- [ ] 日志输出不包含敏感数据
- [ ] 错误信息不泄露敏感路径
- [ ] skill 内容中的凭证必须脱敏

---

## 二、代码安全

### 必须检查

- [ ] 无恶意代码（eval, exec, subprocess 调用外部不可信输入）
- [ ] 无后门
- [ ] 无隐藏的远程执行

### 路径遍历防护

- [ ] skill 文件路径验证（禁止 `..` 攻击）
- [ ] 文件写入限制在 `~/.hermes/skills/` 目录内

### 内容大小限制

- [ ] 单个 skill 文件不超过 100KB
- [ ] skill 内容总量有限制

---

## 三、安全审查流程

### 审查时机

1. **创建 skill 时**（Phase 2/3）
2. **编辑 skill 时**（Phase 3）

### 审查项

```
敏感信息检测 → 恶意代码检测 → 路径遍历检测 → 大小限制检查
```

### 阻断条件

- 发现敏感信息硬编码 → 拒绝写入
- 发现恶意代码 → 拒绝写入
- 路径遍历尝试 → 拒绝并报警

---

## 四、漏洞报告
